# Agents

An agent is a configured AI instance that can process messages, execute tools, and maintain context across conversations. Agents are the core runtime unit in Threaded Stack — they connect an LLM provider to your tools, secrets, and data through a secure, managed pipeline.

## Key Properties

| Property | Description |
|----------|-------------|
| `id` | Unique identifier |
| `name` | Display name |
| `description` | Human-readable description of the agent's purpose |
| `orgId` | Owning organization |
| `config` | Agent configuration (system prompt, model settings, tool bindings) |
| `status` | Current state: active, paused, or archived |
| `createdAt` | Timestamp of creation |

## Agent Configuration

Agents are configured through a flexible `config` object that controls their behavior:

- **System Prompt** — Instructions that define the agent's personality, constraints, and capabilities. The system prompt is sent with every request to the underlying LLM.

- **Provider Routing** — Agents can be linked to one or more providers (OpenAI, Anthropic, custom endpoints). The platform handles API key injection server-side so credentials never leave the backend.

- **Tool Bindings** — Functions and endpoints that the agent can invoke during a conversation. Tools are defined as schemas (name, description, parameters) and executed in sandboxed environments.

- **Model Settings** — Temperature, max tokens, top-p, and other model-specific parameters that control response generation.

```json
{
  "name": "code-reviewer",
  "description": "Reviews pull requests and suggests improvements",
  "config": {
    "systemPrompt": "You are an expert code reviewer. Analyze code changes and provide constructive feedback.",
    "model": "gpt-4",
    "temperature": 0.3,
    "tools": ["analyze-diff", "search-codebase"]
  }
}
```

## Running an Agent

Threaded Stack supports multiple execution modes for agents:

### SSE Streaming

The most common mode for server-to-client streaming. Send a request and receive a stream of server-sent events as the agent processes and responds:

```bash
curl -N -H "Authorization: Bearer tdsk_your_key" \
  -H "Content-Type: application/json" \
  -d '{"message": "Review this function for bugs"}' \
  https://api.threadedstack.app/_/agents/agent_id/run
```

The response is a stream of SSE events including token deltas, tool calls, and completion signals.

### WebSocket Sessions

For bidirectional, real-time communication (such as interactive chat), create a session and connect via WebSocket:

1. Create a session with your agent and org IDs
2. Connect to the WebSocket endpoint with the session token
3. Send and receive messages in real time

See [Authentication](/docs/api-reference/authentication) for details on session token creation and WebSocket connection.

## Agent Lifecycle

Agents follow a straightforward lifecycle:

1. **Create** — Define the agent with a name, description, and initial configuration
2. **Configure** — Link providers, bind tools, set model parameters, and write the system prompt
3. **Test** — Run the agent interactively through the admin dashboard, REPL CLI, or API to validate behavior
4. **Deploy** — Activate the agent for production use by your application or end users
5. **Iterate** — Update configuration, swap providers, or refine the system prompt without downtime

## API Examples

### Create an Agent

```bash
curl -s -X POST \
  -H "Authorization: Bearer tdsk_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "support-agent",
    "description": "Handles customer support inquiries",
    "config": {
      "systemPrompt": "You are a helpful customer support agent.",
      "model": "gpt-4"
    }
  }' \
  https://api.threadedstack.app/_/agents
```

### List Agents

```bash
curl -s -H "Authorization: Bearer tdsk_your_key" \
  https://api.threadedstack.app/_/agents
```

### Run an Agent

```bash
curl -N -X POST \
  -H "Authorization: Bearer tdsk_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "threadId": "thread_id_here",
    "message": "What is the status of my order?"
  }' \
  https://api.threadedstack.app/_/agents/agent_id/run
```
