# Organizations

An organization is the top-level container in Threaded Stack. It serves as the workspace where teams collaborate, and all resources — projects, agents, threads, secrets, and API keys — belong to exactly one organization.

## Key Properties

| Property | Description |
|----------|-------------|
| `id` | Unique identifier |
| `name` | Display name for the organization |
| `slug` | URL-friendly identifier (unique across the platform) |
| `settings` | Organization-level configuration |
| `subscriptionTier` | Current plan: free, basic, developer, or pro |
| `createdAt` | Timestamp of creation |

## Organization Hierarchy

Organizations provide a structured hierarchy for managing resources:

```
Organization
  |-- Projects
  |     |-- Endpoints
  |     |-- Functions
  |     |-- Secrets (project-scoped)
  |
  |-- Agents
  |     |-- Threads
  |     |     |-- Messages
  |     |-- Functions
  |     |-- Providers
  |     |-- Secrets (agent-scoped)
  |
  |-- Secrets (org-scoped)
  |-- API Keys
  |-- Members (Users + Roles)
  |-- Domains
```

Resources are scoped to the organization that owns them. The platform uses an "Exclusive Arc" pattern for polymorphic ownership — for example, a secret belongs to exactly one of: an organization, a project, an agent, or a provider.

## Membership and Roles

Organizations support three built-in roles:

- **Owner** — Full control over the organization, including billing, member management, and deletion. Every organization has exactly one owner.
- **Admin** — Can manage resources (agents, projects, secrets) and invite members, but cannot change billing or delete the organization.
- **Member** — Can use existing resources and create threads, but cannot modify organization settings or manage other members.

Members are added via email invitations. When an invited user accepts and signs in, they are automatically added to the organization with the specified role.

## Quotas

Each organization has usage quotas tracked across 12 resource dimensions. Limits depend on the subscription tier:

| Resource | Description |
|----------|-------------|
| `projects` | Number of active projects |
| `members` | Number of organization members |
| `endpoints` | Total endpoints across all projects |
| `threads` | Total conversation threads |
| `messages` | Total messages across all threads |
| `functionCalls` | Serverless function invocations |
| `runtime` | Total compute runtime (seconds) |
| `orgSecrets` | Organization-scoped secrets |
| `projectSecrets` | Project-scoped secrets |
| `organizations` | Organizations owned by the billing user |
| `price` | Monthly spend limit |
| `retention` | Data retention period (days) |

Quota usage is tracked in real time. When a limit is reached, new resource creation is blocked until usage is reduced or the subscription tier is upgraded.

## API Examples

### List Your Organizations

```bash
curl -s -H "Authorization: Bearer tdsk_your_key" \
  https://api.threadedstack.app/_/orgs
```

### Create an Organization

```bash
curl -s -X POST \
  -H "Authorization: Bearer tdsk_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Team",
    "slug": "my-team"
  }' \
  https://api.threadedstack.app/_/orgs
```

### Get Organization Details

```bash
curl -s -H "Authorization: Bearer tdsk_your_key" \
  https://api.threadedstack.app/_/orgs/org_id_here
```
