# Quick Start

Get up and running with Threaded Stack in about five minutes. This guide walks you through creating an organization, generating an API key, and making your first agent conversation.

## Prerequisites

- **Node.js 18+** (for local development tools and the REPL CLI)
- **A Threaded Stack account** — sign up at the admin dashboard using GitHub, GitLab, Google, or Vercel

## Step 1: Create Your Organization

After signing in to the admin dashboard, you will be prompted to create your first organization. An organization is the top-level container for all your resources — projects, agents, secrets, and team members.

1. Click **Create Organization**
2. Enter a name and slug (e.g., `my-team`)
3. Select your subscription tier (the free tier is available for getting started)

Your organization is now ready. All resources you create will live under this org.

## Step 2: Create an API Key

API keys let you authenticate programmatically without a browser session.

1. Navigate to **Settings > API Keys** in the admin dashboard
2. Click **Create API Key**
3. Give it a descriptive name (e.g., `dev-cli`)
4. Copy the generated key immediately — it starts with `tdsk_` and will only be shown once

## Step 3: Verify Your Connection

Confirm that your API key works by listing your organizations:

```bash
curl -s -H "Authorization: Bearer tdsk_your_key_here" \
  https://api.threadedstack.app/_/orgs
```

You should receive a JSON response containing your organization. If you get a `401` response, double-check that you copied the full key including the `tdsk_` prefix.

## Step 4: Create an Agent

Create an AI agent with a name, provider, and system prompt:

```bash
curl -s -X POST \
  -H "Authorization: Bearer tdsk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-first-agent",
    "description": "A helpful assistant",
    "config": {
      "systemPrompt": "You are a helpful assistant."
    }
  }' \
  https://api.threadedstack.app/_/agents
```

The response includes your new agent's `id` — save this for the next steps.

## Step 5: Start a Thread

Threads are persistent conversations. Create one linked to your agent:

```bash
curl -s -X POST \
  -H "Authorization: Bearer tdsk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "agentId": "your_agent_id"
  }' \
  https://api.threadedstack.app/_/threads
```

The response contains the thread `id` you will use to send messages.

## Step 6: Send a Message

Add a message to the thread to start a conversation:

```bash
curl -s -X POST \
  -H "Authorization: Bearer tdsk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "role": "user",
    "content": "Hello! What can you help me with?"
  }' \
  https://api.threadedstack.app/_/threads/your_thread_id/messages
```

The agent will process your message and generate a response. You can retrieve the full conversation history at any time:

```bash
curl -s -H "Authorization: Bearer tdsk_your_key_here" \
  https://api.threadedstack.app/_/threads/your_thread_id/messages
```
