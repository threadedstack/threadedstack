# Welcome to Threaded Stack

Threaded Stack is a developer platform that unifies authentication, serverless compute, and secure API proxying into a single cohesive system for building AI agent applications. It acts as the nervous system between AI models and external APIs, databases, and tools — enabling teams to build autonomous software without managing complex cloud infrastructure.

Instead of stitching together separate services for auth, secrets management, serverless functions, and LLM orchestration, Threaded Stack provides all of these as a single, integrated platform with built-in security, multi-tenancy, and usage tracking.

## Key Capabilities

- **Authentication Gateway** — JWT/JWKS validation, API key authentication (`tdsk_*` tokens), and session-based WebSocket auth. The auth proxy sits in front of every request, ensuring secure access without burdening your application code.

- **AI Agent Runtime** — Configure agents with specific models, providers, tools, and system prompts. Run them via SSE streaming or WebSocket connections with built-in provider routing and function/tool execution.

- **Serverless Compute** — Execute untrusted code safely in V8 isolates or Firecracker microVMs. The pluggable sandbox layer supports multiple execution backends without changing your agent or endpoint configuration.

- **Secrets Management** — Store API keys and credentials encrypted with AES-256-GCM. Secrets are injected server-side at runtime so AI agents and client code never see raw credentials.

- **Threads and Memory** — Persistent conversation containers with full message history, metadata, and branching. Fork a thread from any message to explore alternative paths without losing context.

- **Multi-Tenant Design** — Organizations, projects, roles, and quotas provide isolation and governance. Tiered subscription plans control resource limits across 12 tracked dimensions.

## Architecture Overview

Every request flows through a layered architecture designed for security and flexibility:

```
Client -> Caddy (TLS/Load Balancer)
       -> Auth Proxy (JWT/JWKS validation, API key verification)
       -> Backend API (CRUD, AI orchestration, FaaS, proxy engine)
       -> External Services (LLM providers, databases, third-party APIs)
```

**Caddy** handles TLS termination and load balancing. The **Auth Proxy** validates every request using JWT tokens from Neon Auth or `tdsk_*` API keys before forwarding to the **Backend**, which provides the core API for managing organizations, agents, threads, secrets, endpoints, and more. The backend also orchestrates AI agent execution, serverless function calls, and secure proxying to external services.

## Who Is It For?

- **Teams building AI-powered applications** that need managed infrastructure for agent orchestration, tool execution, and conversation persistence without assembling a patchwork of services.

- **Developers who want secure-by-default infrastructure** where secrets stay server-side, auth is handled at the gateway, and sandboxed execution prevents untrusted code from escaping.

- **SaaS builders** who need multi-tenant isolation, usage quotas, and subscription tier management out of the box.
